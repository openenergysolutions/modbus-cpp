modbus::IServerChannel class

Server communication channel for a Modbus slave.

Contents

  • Reference
    • Constructors, destructors, conversion operators
    • Public functions

This class represents a server accepting connections from masters. It is responsible for holding acception the connections, deserializing the PDU in ADU and sending the requests to the sessions.

A IServerSession must be created and added with add_session() in order to actually handle the requests. The server can then be started with start().

An instance of this class can be obtained via IModbusManager::create_server_tcp_channel().

The server can only maintain a limited amount of connections opened at a time. If a new connection is established to the server while the maximum number of connections is already reached, then the oldest connection is closed and the newest connection is accepted. The number of connections can be specified in IModbusManager::create_server_tcp_channel().

Destructor.

Warning

This destructor should never be called from user code. IModbusManager will hold a shared pointer to every channels it has created and will release them in IModbusManager::shutdown().

See also

IModbusManager::shutdown(), shutdown()

void modbus::IServerChannel::shutdown() pure virtual

Closes all the associated sessions.

All the active connections will be gracefully closed.

Note

This method is not usually called by user code, but by IModbusManager::shutdown(). It is still possible to shutdown a channel manually using this method, however please note that the IModbusManager will keep holding a shared pointer to it.

Warning

Channels cannot be restarted, but you can safely create a new one via IModbusManager.
