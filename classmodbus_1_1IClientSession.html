modbus::IClientSession class

Client communication session with a Modbus device.

Contents

  • Reference
    • Constructors, destructors, conversion operators
    • Public functions

This class represents a session for communicatiing with a Modbus device. This is the class you'll be using to send requests to the device and schedule them for periodic polling.

An instance of this class can be obtained via IClientChannel::create_session().

It is possible to have multiple sessions on a single channel. For example, if you are communicating with a gateway, you can have a single channel to the gateway and multiple sessions for each slave devices connected to the gateway. The unit identifier for each device is set in IClientChannel::create_session().

For details on the polling behavior, see IScheduledRequest.

The default timeout is configured in IClientChannel::create_session().

This class provides access to timer in order to implement custom polling strategies. See start(const duration_t& duration, const action_t& action) and start(const steady_time_t& expiration, const action_t& action) for more details.

Warning

All the response handlers are called from background threads created by the IModbusManager. Therefore, all response handlers should not block in any way. Otherwise, you risk having other channels not performing their tasks because the thread is blocked or even deadlocks. Also, be sure to use the appropriate mechanisms to eliminate the risks of race conditions in the user code. See IModbusManager and ResponseHandler for more details.

See also

IModbusManager, IClientChannel

modbus::IClientSession::~IClientSession() virtual defaulted

Destructor.

Warning

This destructor should never be called from user code. IClientChannel will hold a shared pointer to every sessions it has created and will release them in IClientChannel::shutdown(), which is probably called by IModbusManager::shutdown().

See also

IModbusManager::shutdown(), IClientChannel::shutdown(), shutdown()

void modbus::IClientSession::shutdown() pure virtual

Cancels all pending requests and all scheduled requests.

Note

This method is usually not called by user code, but by IClientChannel::shutdown(), which is normally called by IModbusManager::shutdown(). It is still possible to shutdown a session manually using this method, however please note that the IClientChannel will keep holding a shared pointer to it.

Warning

Once a session is shutdown, sending requests through it is not permitted. Sessions cannot be restarted, but you can safely create a new one via IClientChannel.

See also

IModbusManager::shutdown(), IClientChannel::shutdown()
