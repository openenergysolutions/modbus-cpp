template<typename T>
modbus::Expected class

Expected type often used in return values and callbacks.

Template parameters
T Type of the expected value

Contents

  • Reference
    • Public static functions
    • Constructors, destructors, conversion operators
    • Public functions

This type represents a value that is expected to be returned, but may be impossible to obtain. If is_valid() returns true, it means that the value was indeed returned and it may be obtained by get(). Otherwise, it means an error occured and the value could not be returned. In this case, you can determine the type with has_exception() templated with the type of error it may have returned. You can also get the error with get_exception().

It is very similar to Rust's Result type. This class uses the std::exception_ptr union trick to work.

This class is heavily inspired by Andrei Alexandescu Expected class that he presented in 2012 in his talk "Systematic Error Handling in C++" available on Channel9: https://channel9.msdn.com/Shows/Going+Deep/C-and-Beyond-2012-Andrei-Alexandrescu-Systematic-Error-Handling-in-C

This method should be used when the value could not be obtained (i.e. an error occured). Here's an example usage:

Expected<int> foo()
{
     return Expected<int>::from_exception("Could not foo...");
}

template<typename T>
bool modbus::Expected<T>::is_valid() const

Check if expectation is valid.

Returns true if the expectation is valid, false if it contains an error

If this method returns true, the value can be obtained with get(). Otherwise, the error can be obtained with get_exception().
