enum class modbus::ExceptionType

Modbus exception type.

See section 7: MODBUS Exception Responses for more details about the exception.

Enumerators
IllegalFunction

0x01 Illegal function

Function is not allowed on the slave, or the slave is not in a state where it can respond to this type of request.
IllegalDataAddress

0x02 Illegal data address

The address range does not fit in the device mapping.
IllegalDataValue

0x03 Illegal data value

A value in the query is not valid (e.g. length not set properly). It does not mean the value is outside the expectation of the application program.
ServerDeviceFailure

0x04 Server device failure

An unknown error occured on the slave device.
Acknowledge

0x05 Acknowledge

The request was accepted, but it requires a long processing type.

Used with programming commands (not supported by this library)
ServerDeviceBusy

0x06 Server device busy

The server is already processing a long-duration program.

Used with programming commands (not supported by this library)
MemoryParityError

0x07 Memory parity error

The extended file area failed to pass a consistency check.

Used with function codes 0x20 and 0x21 (not supported by this library)
GatewayPathUnavailable

0x0A Gateway path unavailable

When use with a gateway, the gateway could not map the input port to the output port.
GatewayTargetDeviceFailedToRespond

0x0B Gateway target device failed to respond

When use with a gateway, no response was received from the target device.
Unknown

Unknown Modbus exception.

Notify the vendor it should not respond with this message.

template<typename TResponse>
using modbus::ResponseHandler = std::function<void(const Expected<TResponse>&)>

Response handler used in callbacks.

The response handler will be called with the expected response. Be sure to check that the expectation is valid with Expected::is_valid() before using it.

Warning

All the response handlers are called from background threads created by the IModbusManager. Therefore, all response handlers should not block in any way. Otherwise, you risk having other channels not performing their tasks because the thread is blocked or even deadlocks. Also, be sure to use the appropriate mechanisms to eliminate the risks of race conditions in the user code.
